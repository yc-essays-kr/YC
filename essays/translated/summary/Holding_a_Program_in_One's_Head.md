# # Holding a Program in One's Head

**프로그래밍에서 코드를 머릿속에 담는 것의 중요성**

우수한 프로그래머는 수학자가 문제를 머릿속에서 다루듯이 자신의 코드 전체를 머릿속에 담고 자유자재로 조작할 수 있어야 한다. 이는 특히 프로젝트 초기에 중요한데, 문제 해결 방식뿐만 아니라 문제 자체를 바꿀 수 있는 유연성을 제공하기 때문이다.

하지만 프로그램을 머릿속에 완전히 로드하는 것은 쉽지 않다. 몇 달 간 프로젝트를 떠나 있으면 다시 이해하는데 며칠이 걸리고, 매일 작업을 시작할 때도 30분 정도의 시간이 필요하다. 일반적인 사무환경의 개발자들은 이런 상태에 거의 도달하지 못한다.

이를 개선하는 방법들로는: ①주의 산만을 피하기(특히 예정된 방해는 더 나쁘다), ②긴 시간 연속으로 작업하기, ③간결한 프로그래밍 언어 사용하기, ④지속적인 코드 리팩토링, ⑤읽기 쉬운 코드 작성하기 등이 있다. 이러한 방법들을 통해 프로그래머는 코드를 더 깊이 이해하고 효과적으로 다룰 수 있게 된다.
