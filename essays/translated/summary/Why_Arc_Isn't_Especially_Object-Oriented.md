# # Why Arc Isn't Especially Object-Oriented

**에세이 요약: "Arc가 객체지향적이지 않은 이유"**

저자는 현재 객체지향 프로그래밍(OOP)에 대한 과도한 열풍을 비판하며, OOP가 모든 프로그램에 필수적이지 않다고 주장한다. 

사람들이 OOP를 선호하는 5가지 이유 중 3.5개가 잘못되었다고 분석한다: 1) 정적 타입 언어의 한계를 우회하는 수단, 2) 대기업에서 평범한 프로그래머들을 관리하기 위한 도구(하지만 코드 비대화 초래), 3) 많은 작업을 하는 것처럼 보이게 하는 도구, 4) 언어 확장 가능성(하지만 더 나은 방법 존재), 5) 특정 도메인(시뮬레이션, CAD 등)에서의 유용성.

저자는 개인적으로 OOP가 필요했던 적이 없으며, Common Lisp의 강력한 객체 시스템도 사용한 적이 없다고 밝힌다. Lisp의 해시 테이블과 클로저 조합으로 다른 언어에서 OOP로 해결할 문제들을 처리했다고 설명한다. 자신의 경험에만 기반한 언어 설계의 위험성을 인정하면서도, 불필요한 기능을 추가하는 것이 더 위험하다고 결론짓는다.
