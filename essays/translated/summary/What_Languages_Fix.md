# # What Languages Fix

**프로그래밍 언어들이 해결하고자 한 문제들**

이 에세이는 Kevin Kelleher의 아이디어를 바탕으로 각 프로그래밍 언어가 해결하려고 했던 기존 언어의 문제점들을 간결하게 정리한다.

주요 패턴들을 보면: **낮은 수준의 언어 문제**(Algol, C는 어셈블리어가 너무 저수준이라는 문제 해결), **데이터 타입 부족**(Pascal, PL/1), **복잡성과 혼란**(Java는 C++의 복잡함, Python과 Ruby는 Perl의 복잡함 해결), **특정 도메인 한계**(APL은 배열 조작, Prolog는 논리 프로그래밍), **기업 통제 문제**(C#은 Sun의 Java 통제, Java는 Microsoft 위협 대응) 등이 있다.

흥미롭게도 많은 언어들이 이전 언어를 "너무 복잡하다(kludge)" 또는 "무섭다(scary)"고 평가하며 등장했다. 이는 프로그래밍 언어 발전사가 기존 언어의 한계와 문제점을 해결하려는 지속적인 시도의 연속이었음을 보여준다.
