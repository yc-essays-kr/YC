# # Succinctness is Power

**핵심 요약:**

이 에세이는 "간결함이 곧 힘"이라는 가설을 제시합니다. 저자는 프로그래밍 언어의 목적이 고수준 언어로 10줄에 표현할 것을 기계어로는 1000줄이 필요한 상황에서 레버리지를 얻는 것이라고 주장합니다. 따라서 소스 코드를 더 작게 만드는 것이 고수준 언어의 핵심 가치이며, 언어의 힘은 프로그램을 얼마나 작게 만들 수 있는지로 측정된다고 봅니다.

코드 크기 측정에서 일반적인 '코드 라인 수'나 '문자 수'보다는 '요소의 개수'가 더 나은 지표라고 제안합니다. 여기서 요소란 소스 코드를 트리로 표현했을 때 각각의 노드가 되는 것들(변수명, 함수명, 숫자, 텍스트, 블록 등)을 의미합니다. 이러한 트리의 크기가 프로그램을 이해하고 작성하는 데 필요한 정신적 작업량과 비례한다고 설명합니다.
