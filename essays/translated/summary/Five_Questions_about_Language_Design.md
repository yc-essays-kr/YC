# # Five Questions about Language Design

**프로그래밍 언어 설계에 관한 5가지 질문 요약**

이 에세이는 프로그래밍 언어 설계의 핵심 원칙들을 제시합니다.

**주요 내용:**
1. **인간을 위한 설계**: 프로그래밍 언어는 컴퓨터가 아닌 사람을 위해 존재한다. 의자 설계처럼 인간의 한계를 고려해야 하며, 수학적 우아함보다는 인간의 약점을 보완하는 것이 중요하다.

2. **자신과 동료를 위한 설계**: 역사적으로 최고의 언어들은 설계자가 직접 사용하기 위해 만들어졌다. 다른 사람(특히 실력이 부족한 프로그래머)을 위해 설계된 언어들은 대부분 실패했다.

3. **프로그래머에게 최대한의 제어권 제공**: 많은 언어들이 가정교사처럼 프로그래머를 제한하려 하지만, 반대로 최대한의 제어권을 주는 것이 바람직하다. Lisp처럼 프로그래머를 동등한 파트너로 대우하는 접근이 효과적이다.

4. **간결성 추구**: 해커들은 본질적으로 간결함을 사랑한다.

**핵심 메시지:** 좋은 프로그래밍 언어는 최고의 프로그래머들이 선호하는 언어이며, 인간의 한계를 이해하면서도 최대한의 자유와 제어권을 제공해야 한다.
