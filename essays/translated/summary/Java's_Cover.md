# # Java's Cover

**Java's Cover 요약**

이 에세이는 Java 언어 자체에 대한 비판이 아니라 해커의 '기술 직감'에 대한 사례 연구입니다. 작가는 Java를 직접 사용해본 적은 없지만, 여러 신호를 통해 Java가 성공적이지 못할 언어라고 판단했다고 말합니다.

그 이유로 6가지를 제시합니다: ①과도한 홍보 마케팅(진짜 표준은 저절로 확산됨), ②낮은 목표 설정(C 프로그래머들이 쉽게 배우도록 의도적으로 단순화), ③불순한 동기(Sun이 Microsoft를 견제하기 위한 수단), ④사랑받지 못함(다른 언어들과 달리 Java를 사랑한다는 개발자를 본 적이 없음), ⑤강제적 사용(자발적 선택이 아닌 외부 압력으로 사용), ⑥너무 많은 개발자 참여(좋은 언어들은 소규모 그룹이 개발했는데 Java는 위원회가 운영).

작가는 출판업계가 책 표지로 내용을 알 수 있게 발전한 것처럼, 기술도 겉모습으로 판단할 수 있다고 주장하며, 해커들이 기술의 좋고 나쁨을 직감적으로 판단하는 능력에 대해 논의합니다.
