# # The Hardest Lessons for Startups to Learn

**스타트업이 배우기 어려운 교훈들 (2006년 4월)**

폴 그레이엄이 투자한 스타트업들을 관찰하며 발견한 핵심 교훈들입니다.

**1. 빠른 출시 (Release Early)**
완벽한 제품보다는 최소 기능 제품(MVP)을 빠르게 출시해야 합니다. 사용자들은 버그는 싫어하지만 기본적인 버전 1은 용인합니다. Reddit처럼 성공한 서비스도 사용자가 누군지 정확히 모르기 때문에, 추측보다는 실제 출시 후 피드백을 받는 것이 중요합니다. Wufoo는 데이터베이스 완성 전에 폼 빌더를 먼저 출시해 8만 3천명의 사용자로부터 Linux 호환성 문제를 발견했습니다. 빠른 출시는 근본적인 문제를 조기에 발견하게 하고, 더 긴박감 있게 일하게 만듭니다.

**2. 지속적인 기능 개발**
초기 출시 후에도 꾸준히 사용자 경험을 개선하는 기능들을 추가해야 합니다. 매일 조금씩이라도 시스템을 개선하는 것이 스타트업의 생명력입니다.
