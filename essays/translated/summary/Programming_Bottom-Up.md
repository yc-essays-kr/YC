# # Programming Bottom-Up

**Programming Bottom-Up 요약**

이 에세이는 전통적인 하향식(top-down) 프로그래밍과 대비되는 상향식(bottom-up) 프로그래밍 접근법을 설명합니다.

**전통적 방식**: 큰 프로그램을 여러 주요 서브루틴으로 나누고, 각각을 다시 세분화하여 이해 가능한 단위로 만드는 하향식 설계

**상향식 방식**: 문제에 맞게 프로그래밍 언어 자체를 확장하는 접근법. 필요한 연산자나 기능이 없으면 직접 만들어 언어를 프로그램에 맞춰 발전시킴. 언어와 프로그램이 함께 진화하여 마치 그 문제를 위해 언어가 설계된 것처럼 보이게 됨.

**상향식 설계의 장점**: 
- 더 작고 효율적인 프로그램 생성
- 코드 재사용성 향상  
- 가독성 개선
- 컴포넌트 수 감소로 인한 오류 가능성 축소

핵심 메시지는 언어를 문제에 맞춰 확장함으로써 더 명확하고 간결하며 효율적인 코드를 작성할 수 있다는 것입니다.
